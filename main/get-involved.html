Get involved ​

Platform Mesh is an open source effort and welcomes contributions from the community. The projects developed in context of Platform Mesh have dedicated community calls, chat channels, and repositories.

Platform Mesh ​

Discussions around the Platform Mesh as a whole happen in the Linux Foundation Zulip chat channel:

neonephos-platform-mesh-discussion

kcp ​

kcp is the Service Control Plane for Platform Mesh. It is an open source project hosted by the CNCF.

The best starting point to get in touch is the #kcp-users channel on Kubernetes Slack and the biweekly community calls.

For more information on getting involved with kcp, please see the Getting in Touch section of the kcp documentation.

api-syncagent ​

api-syncagent is an operator that allows publishing CRDs from a control plane to APIExports in a kcp workspace. It is part of the kcp project.

kube-bind ​

kube-bind is a project that enables providers to offer their APIs to consumers across distinct control planes using the Kubernetes Resource Model.

The best starting point to get in touch is the #kube-bind channel on Kubernetes Slack.

For more information on getting involved with kube-bind, please see the Getting in Touch section of the kube-bind documentation.

multicluster-runtime ​

multicluster-runtime is a library to write Kubernetes controllers that reconcile resource across Kubernetes-like control planes. The project is hosted by SIG Multicluster.

Discussions happen in the #sig-multicluster channel on Kubernetes Slack.
